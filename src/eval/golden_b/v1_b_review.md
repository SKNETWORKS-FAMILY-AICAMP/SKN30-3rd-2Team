> 목적: 라벨 없는 M:N 커버리지 결과를 읽고 **무엇이 잘 되고 무엇이 안 되는지**, 그 **원인**과

## 0. 이 문서 읽는 법 (지표 빠르게)

- **coverage** `= (전체 표준 − MISSING) / 전체 표준` — 문서가 그 유형 표준조항 중 **몇 %에 대응 조항을 가졌나**. 검색(rerank sigmoid ≥ `match_threshold`)이 표준을 찾아 물면 "대응 있음", 못 물면 MISSING.
- **best-fit** — 커버리지가 가장 높은 유형. 문서의 유형을 라벨 없이 **추정**한 것.
- **CHANGED / NONE** — 대응 표준을 찾은 뒤, 본문 글자 일치율(`difflib`)이 `change_threshold`(0.85) 이상이면 NONE(변경 없음), 미만이면 CHANGED(변경됨).
- ⚠️ 라벨(정답)이 없으므로 coverage 의 MISSING 은 **'진짜 누락'인지 '매칭 실패'인지 구분하지 못한다.**

## 1. 한 줄 요약

- **평가는 정상 실행됐다.** 파서 수정(제N조 라인경계 분해) 후 5문서 전부 10~15건으로 파싱되고 매트릭스가 나왔다(직전 실행은 파싱 0건 축퇴였음).
- **✅ 유형 판별(검색 라우팅)은 성공이다.** 5문서 전부 best-fit=`SW_FREELANCE`(상식 부합), SW 커버리지가 SI/SM 의 **약 5배**. 실문서에서도 RAG 검색이 올바른 유형 코퍼스로 라우팅한다.
- **⚠️ 이탈 분류는 여전히 고장(축퇴)이다.** 모든 문서의 **모든 조항이 CHANGED**(NONE=0, EXTRA=0). 이는 Track A(`../golden/v1_review.md` §2 원인 A)에서 본 축퇴가 **합성 아티팩트가 아니라 실문서에서도 재현**됨을 뜻한다.

## 3. 핵심 발견 — 이탈 분류 축퇴가 실문서에서도 (가장 중요)

**모든 조항이 CHANGED 로 찍힌다.** 5문서·65개 조항 전부 CHANGED, NONE 0건.

**원인(= Track A 원인 A 와 동일):** NONE/CHANGED 를 **글자 단위 일치율**(공백 제거 후 `difflib`, 0.85 기준)로 가르는데, 사용자 조항(단문)과 표준 조항(조 전체·여러 항)은 **길이·형식이 크게 달라** 내용이 같든 다르든 일치율이 0.85 를 절대 못 넘는다. 그래서 매칭된 조항은 죄다 CHANGED.

**Track B 가 보태는 정보:** 이 축퇴가 **합성 골든셋 라벨 문제가 아니라 파이프 자체의 한계**임을 실계약으로 확증했다. 즉 실사용자가 계약서를 올리면 **모든 조항이 "변경됨"으로 떠서** triage(어디를 봐야 하나) 신호를 못 준다. 안전 측면(FN=0, 놓치지 않음)은 유지되나 **유용성**이 무너진다.

> 이건 커버리지 지표의 결함이 아니라 **deviation 분류기의 결함**이다. 커버리지(검색 라우팅)는 정상이고, 그 위의 NONE/CHANGED 판정만 축퇴다.

## 4. 커버리지 해석 & 주의

- **coverage 0.35~0.52 는 그럴듯하다.** 짧은 실계약(10~15조)이 23조짜리 SW 표준의 **절반가량**에만 대응 조항을 가진다. 나머지(11~15조)는 MISSING — 실계약이 실제로 없는 조항이거나 매칭 실패이거나(라벨이 없어 구분 불가).
- **분리비는 deviation 이 아니라 검색이 만든다.** 모든 조항이 CHANGED 인데도 유형 분리가 뚜렷한 건, coverage 가 **rerank 매칭(sigmoid ≥ `match_threshold`)** 으로 결정되기 때문. 즉 이 매트릭스가 재는 것은 "**검색이 올바른 유형을 찾는가**"지 "이탈을 맞히는가"가 아니다. 지표의 유효 범위를 이렇게 못 박아 둔다.
- **절대값을 목표로 삼지 말 것.** 5건·SW 편중·라벨 없음 → 커버리지 절대값은 방향 참고. v2 는 **같은 5문서에 시스템만 바꿔** 돌린 뒤 델타로 본다.

## 5. 트랙 B(이 평가 설계) — 강점·약점

**강점**
- 라벨 0개로 **실문서 강건성 + 유형 판별 + 버전 비교** 신호를 결정론적으로 뽑았다(LLM-judge 없음, 규칙 #5 준수).
- 순환 편향 위험이 원천적으로 없다(정답 라벨을 시스템 출력에서 만들지 않음).
- 실제 제품 파싱 버그를 노출하는 1차 목적을 달성.

**약점**
- **표본이 SW/용역 도메인에 편중(5/5).** SI/SM 실문서가 없어 판별력 검증이 **일방향**이다 — "SW 문서가 SW 로 가는가"만 봤고, "SI 문서가 SI 로 가는가"는 못 봤다.
- **best-fit 이 '가장 덜 나쁜 fit'일 수 있다.** test_sunny10·동행용역까지 SW(0.52)로 붙는데, 절대값 0.52 는 도메인 소속을 **단정할 만큼 높지 않다.** 이 둘이 진짜 SW 계약인지 커버리지만으론 못 가린다(사람 확인 필요).
- **coverage 절대값이 true-missing 과 match-failure 를 섞는다**(설계상 알려진 한계).
- **deviation 분포가 전부 CHANGED** 라, 커버리지 외 정보(이탈 품질)를 지금은 못 준다.

## 6. v2(시스템 버전)에서 볼 것 — 체크리스트

트랙 B 의 vN 은 시스템 버전이므로, 아래를 바꾼 뒤 **같은 5문서**를 재실행해 커버리지·분리비 **델타**로 확인한다.

- [ ] **deviation 축퇴 해소(최우선, Track A 와 공통 미결):** 비교 단위를 조↔조가 아니라 **항↔항**으로 맞춰 NONE/CHANGED 판정. 이게 풀려야 CHANGED 분포가 의미를 가진다. (부정어·금액·당사자 변화는 별도 명시 검출)
- [ ] **SI/SM 실문서 확보** → 판별력 양방향 검증(SI 문서가 SI 로 best-fit 되는가). 현재 최대 공백.
- [ ] **match score 분포를 커버리지에 병기** → "약한 매칭(경계선)"과 "강한 매칭"을 구분해, coverage 0.52 가 견고한 대응인지 확인.
- [ ] **best-fit 신뢰 하한 표기:** 최상위 유형 커버리지가 절대적으로 낮으면(예: <0.3) "유형 판별 불가"로 표식 — 스코프 밖 문서 방어.
- [ ] (리랭커/임계값 튜닝 시) 재실행 → `v2_b_result.md` 생성 → v1 대비 SW 커버리지·분리비 변화 기록.
