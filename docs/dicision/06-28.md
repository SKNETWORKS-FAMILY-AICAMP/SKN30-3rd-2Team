- **역할:** 리드 = 총괄·구현, PM(LLM AGENT) = 기반/규격/문서·일 분배. 팀원 4명은 **모듈 단위**로 분담.

## 1. 아키텍처
- **헥사고날 채택:** `contracts`(동결 계약) ← `core`(순수함수) · `adapter`(외부 I/O) → `pipe`(조립) → `mcp_server`/`demo`.
- **core는 adapter를 직접 import하지 않음** — 외부 작업은 **포트(Protocol)로 주입**. → 순수 → TDD 용이.
- **이유:** 비전공 팀원이 동결 계약에만 의존해 병렬 작업, fake 주입으로 독립 테스트 가능.

## 4. 포트 구현체 배치
- **결정: 구현체를 두 곳에 둔다.**
  - **`adapter/` 기본구현:** 단일 도구가 포트와 그대로 일치 → `Retriever`=`VectorManager`(`vector`), `Embedder`=`Bgem3Embedder`(`embedder`).
  - **`contracts/implement/` 조합 구현:** 여러 어댑터+변환을 엮어야 할 때 → `Parser`=`KordocParser`(kordoc+splitter), `Grounder`=`KoreanLawGrounder`(koreanLaw+`CATEGORY_QUERIES`+파싱). `Graph`=`ClauseGraph`(예정).
  - **동결 영역(enums·models·ports)은 순수 유지**, `implement/`는 동결 아님(포트 시그니처만 준수).
- **Grounder 매핑:** `category → 법령 검색어`를 코드 상수(`CATEGORY_QUERIES`)로 보유, korean-law 응답 텍스트를 `GroundingLaw`로 파싱.

## 5. 파이프라인 & 도구 선택
- **청킹 도구: LlamaIndex `MarkdownNodeParser`** 채택 (`llama-index-core` 의존 추가).
- **조항 분해 배치 결정:** **순수 마크다운 split은 어댑터**(`adapter/markdown_splitter.py`, `splitter`), **`제N조` 해석 등 도메인 로직은 `pipe/normalize`**.
  - **이유:** 도메인 무관 split을 표준 코퍼스(normalize)와 사용자 업로드(parse_contract)가 **공유**. 어댑터에 도메인 로직 금지.
- **숫자 프리픽스 파일 규칙:** `2.normalize.py`·`3.build_index.py`는 import 불가 → **로직은 import 가능한 모듈**(`pipe/normalize.py`·`pipe/build_index.py`)에, **숫자 파일은 CLI 래퍼**.
- **명령:** `just build-db`(migrate→build-index 원클릭), `just migrate`/`build-index`/`normalize`/`parse`.

## 6. TDD & 테스트 전략
- **테스트가 곧 규격:** 모듈 규격을 `tests/`에 먼저 고정, 팀원은 통과시키며 구현.
- **미구현 모듈은 skip-마킹된 스펙 테스트** + **스켈레톤 스텁**(시그니처+docstring+`NotImplementedError`) 제공.
  - **이유:** 비전공 친화 — IDE 빨간줄 제거, "파일 열어 본문 채우고 skip 한 줄 지우면 끝".
- **pytest 설정:** `pythonpath=["src","."]`, `testpaths=["tests"]`. `pipe`·`eval`을 패키지화.
- **core 4종**(matching/deviation/graph/toxic)은 현재 동작을 **회귀 고정**(통과).

## 7. 평가(eval) & RAGAS
- **RAGAS 미사용(1차).** 이유: ① LLM-judge라 동결 규칙 #5 위반, ② 1차엔 평가할 생성물 없음, ③ 우리 태스크는 검색+분류라 고전 IR 지표가 정확히 맞음. → **2차(생성 레이어) 후보로만** 남김.
- **지표:** 검색=`Recall@k`/`MRR`, 이탈=`precision/recall`, **ablation(8.5)=4변형 비교(BM25/dense/hybrid/+reranker)** = 대표 결과물.
- **구조·재사용:** `ablation → run_eval → metrics` (중복 구현 금지). 순수 집계와 CLI(검색 실행)를 분리.
- **driver:** 골든셋 `user_clause` → `vector.search` → `cases` 만드는 글루는 `run_eval`/`ablation`의 CLI부(통합).
- **골든셋:** (사용자 조항 ↔ 정답 표준조항) + 이탈 라벨 + `trap`(함정)·`gold_toxic`. `MISSING`은 **계약 단위**로 평가.

## 11. 추가 의사결정 (실시간 세션 피드백)
- **법제처 API 인증키 환경변수명 전면 통일 및 연동**: 프로젝트의 공식 API 키 환경변수명을 **`OPEN_LAW_API_KEY`**로 명확히 통일하였습니다. 다만 외부 MCP CLI(`korean-law`)가 요구하는 `LAW_OC` 변수와의 상호 호환을 위해, `just check-law-key` 가동 시 두 키를 모두 `.env`에 자동 저장하고 복사용 가이드를 제공하여 호환성을 강화했습니다.
- **`Bgem3Embedder.compute_similarity` 구현 및 코사인 유사도 연산 표준화**: `ports.Embedder` 프로토콜에 규정되어 있으나 어댑터 구현체에서 누락되어 있던 유사도 계산 함수를 구현했습니다. 임베딩이 이미 L2 정규화(`normalize_embeddings=True`) 되었음을 활용해, 외부 라이브러리(NumPy) 없이 순수 파이썬 루프 기반의 벡터 내적(Dot Product)을 통해 코사인 유사도를 계산하는 방식으로 표준화했습니다.
- **IDE 내 문서 팝업 힌트(Hover Documentation) 강화**: `src/contracts/` 내의 `models.py`, `ports.py`, `enums.py`에 작성되어 있던 일반 코드 주석(`#`)을 변수 바로 아래에 배치하는 삼중 따옴표(`"""`) 공식 어트리뷰트 독스트링(Attribute Docstring) 형식으로 전면 변경하여 IDE 가독성을 극대화하기로 결정했습니다.
- **`Grounder` 카테고리 매핑 법률 조항 팩트체크 및 정정**: `korean-law-mcp`를 실시간 터미널 쿼리하여 법적 정합성을 대조하고, 기존 매핑 오류였던 합의관할(민사소송법 제2조 ➔ 제29조) 및 완성전 계약 해제(민법 제668조 ➔ 제673조)를 명확하게 정정하여 신뢰도를 확보했습니다.
