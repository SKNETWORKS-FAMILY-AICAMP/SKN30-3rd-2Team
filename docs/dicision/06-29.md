- **granularity 추가도 동결 변경이다:** "재사용이라 승인 불필요"가 **아님**. (a)granularity·(b)change_kind·(c)ElementResult **3건을 쪼개지 말고 한 묶음으로 일괄 승인**. 재사용 논거는 "승인 생략"이 아니라 "같은 승인표 안에서 더 나은 형태(가법·하위호환)를 추천안으로 제시하라"는 뜻.

## 3. 정렬 방식 — "다대다 헝가리안" 모순 정정
- **결정: 원안의 "다대다 최적 정렬 + `scipy linear_sum_assignment`(헝가리안)"은 모순이라 폐기.** 헝가리안은 **1:1 할당**이라, 하도급 본문에서 가장 흔한 **1:N**(표준 1항을 사용자가 여러 문장으로 풂)·**N:1**(여러 항을 한 문장에 뭉침)을 1:1 로 풀면 잔여를 강제 MISSING/EXTRA 로 떨궈 **잡으려던 오탐을 역으로 생성**.
- **채택: 임계값 기반 다대다 매칭** — 각 표준 element 에 임계값 이상 사용자 명제를 모두 연결(1:N·N:1 자연 허용). 코퍼스가 작아 최적성보다 **재현성·단순성** 우선.
- **대안(보류): 1:1 최적 정렬 + 잔여 흡수 패스** — 헝가리안 1차 할당 후 임계값 위 잔여를 2차 그리디로 흡수. 복잡도 높아 기본값 아님.

## 4. NLI 보조 신호 — 규칙 #1 경계 확정
- **결정: `KLUE-NLI` 는 신호 1개로만 사용**, 최종 불리/심각 판정은 양태·당사자·수치 **룰**로 내린다.
- **이유:** NLI 추론 자체는 결정론적 분류라 규칙 #1("분류만" 허용)·#5(LLM-judge 아님) 형식상 통과. **그러나 "Contradiction → 불리한 변경" 매핑**은 규칙 #1 이 금지한 "불리함 판단 생성"에 닿는 **경계**.
- "Contradiction→불리" 매핑 채택 여부는 **사람 확인 대상**으로 남기고, **Ablation 측정으로 기여 입증될 때만** 활성화. (전날 "규칙 레버 먼저, NLI 는 측정 정당화 시" 방향과 일관·강화)

## 5. `protects` → `beneficiary` 재정의
- **결정: 의미 모호한 `protects`(SUBCONTRACTOR/PRINCIPAL) 폐기 → `beneficiary: SUBCONTRACTOR | PRINCIPAL | MUTUAL`(이 항의 이익이 누구에게 귀속되는가).**
- **이유:** "보호 의도"와 "수혜 당사자"는 다른 축. PRINCIPAL 이 값에 있으면 "원사업자를 보호하는 조항?"이라 해석 불가. 5단계 심각도 연산이 성립하려면 한 줄 정의가 필요.
- **"불리함" 정의:** `beneficiary == SUBCONTRACTOR` 이고 `polarity == PROTECTIVE` 인 element 의 누락(MISSING) 또는 약화(CHANGED/WEAKENED).

## 6. 심각도 — 추상 "태그 연산" → 결정론 결정표
- **결정: 5단계 심각도를 `(required, beneficiary, deviation/change_kind) → severity` 고정 룩업표로 산출.** 추상어 금지.

## 7. 런타임 분해 재현성 — `kss` 표현 정정
